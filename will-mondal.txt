   1. What is Semantic HTML? 
   it comprises tags which provide information for what they comprise, for example <h1> tag will include text that makes up the title. Some tags like <div> have no meaning by themselves.
    2. What is HTML used for? 
    it is a markup language and used to provide the raw content (mainly text and images) of a webapge with no styling.
    3. What is an attribute and where do we put it? 
    an attribute is an identifier that goes inside the opening tag and can be called in CSS. 
    4. What is the h1 tag used for? How many times should I use it on a page?
    it should be used as the title of your webpage and therefore only used once. 
    5. Name two tags that have required attributes
    anchor tag requires href, and image tag requires src 
    6. What do we put in the head of our HTML document? 
    in the head goes things that are useful to developers such as meta tags, but the user will not actually see any. 
    7. What is an id? 
    an id is an attribute that labels an element with a specific string
    8. What elements can I add an id to? 
    you can add an id to any element
    9. How many times can I use the same id on a page? 
    once
    10. What is a class? 
    a class is also an identifying attribute but with less specificity than id
    11. What elements can I add a class to? 
    any
    12. How many times can I use the same class on a page?
    as much as you like  
    13. How do I get my link to open in a new tab?
    within the opening tag after the href link, add target = "_blank"
    14. What is the alt attribute in the image tag used for? 
    it is used as a substitute descriptor for an image for those using a screen reader
    15. How do I reference an id?

    16. What is the difference between a section and a div
    section is used to divide different parts of the website up whilst div is used more to ensure a new line by creating a block-level element.
    17. What is CSS used for?

    37. Walk me through Lambda's git flow. 
    Fork the repo
    clone it by putting git clone and the url of the repo into terminal
    cd to the directory you are working on 
    make a new personal branch with git checkout -b "firstname-lastname"
    make all the changes you need to 
    git add . all of your changes 
    then commit your changes to (primes them for sharing) with git commit -m"and include a descriptive message of changes you've made"
    then finally push them back up to github with git push, if it is the first time your branch is being used you will need to write git push -u origin *your branch name*
    finally create a pull request making sure the right repository (yours) is main
